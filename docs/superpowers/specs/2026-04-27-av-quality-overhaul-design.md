# AV Quality Overhaul — Design Spec

## Problem

Videos generated by the pipeline are too short (15-30s when 60s+ is needed), visually repetitive (same 4 scenes, same animations, hard cuts), and the voiceover duplicates on-screen text instead of complementing it. ProductShort has only 4 scene types; all videos look identical.

## Goals

1. Minimum 60s duration for ProductShort, 90s for tutorials
2. Voiceover complements visuals (show vs tell protocol)
3. Expand ProductShort from 4 to 12 scene types
4. Activate `@remotion/transitions` for cinematic scene changes
5. Rewrite all 3 agent prompts (copywriter, director, audio_planner)

## Non-goals

- Custom fonts (system-ui stays)
- Particle systems, 3D effects, audio visualization (future scope)
- New music tracks (out of scope for this iteration)

---

## 1. Duration and Content Density

### Rules

| Format       | Min duration | Min scenes | Max scenes | Scene duration |
| ------------ | ------------ | ---------- | ---------- | -------------- |
| ProductShort | 60s          | 6          | 12         | 5-12s each     |
| Tutorial     | 90s          | 8          | 20         | 5-15s each     |

### Mandatory narrative structure (ProductShort)

| Block    | Purpose                         | Duration | Scenes                                             |
| -------- | ------------------------------- | -------- | -------------------------------------------------- |
| Hook     | Grab attention, provocation     | 5-8s     | 1 (hero)                                           |
| Context  | Problem or current situation    | 8-12s    | 1-2 (problem-solution, quote, callout)             |
| Solution | Demonstration or arguments      | 15-20s   | 3-4 (benefits, bullet-slide, screenshot, terminal) |
| Proof    | Data, comparisons, testimonials | 8-12s    | 2 (stat-reveal, comparison-table, timeline)        |
| Close    | Price + call to action          | 5-8s     | 1-2 (pricing, cta)                                 |

Total: 41-60s of content minimum, plus transitions.

### Enforcement

The copywriter prompt will include a duration calculation step:

1. Sum all `durationInSeconds` values
2. If total < minimum, add scenes from underweight blocks
3. Never pad with empty time; add real content

## 2. Voiceover: Show vs Tell Protocol

### Problem

The audio_planner writes voiceover text per scene but doesn't know what text is already visible. The voice repeats what the viewer is reading.

### Solution: Internal annotation fields

The copywriter annotates each scene with two internal fields (not rendered):

```json
{
  "type": "benefits",
  "items": [{ "text": "24/7 cobertura" }],
  "visualText": "24/7 cobertura, sin permanencia, precio garantizado",
  "narrativeIntent": "Explain WHY these benefits matter emotionally"
}
```

### Rules for audio_planner

1. **Never repeat `visualText`.** If screen says "24/7 cobertura", voice says "Porque un accidente no avisa"
2. **Voice contextualizes, screen synthesizes.** Visual = the data. Voice = why it matters
3. **Mandatory silence on impact scenes.** hero, pricing, stat-reveal: minimum 2s without voice
4. **Timing offset:** Voice starts BEFORE visual (anticipates) or AFTER (reinforces). Never exactly synchronized
5. **Tone:** Conversational, not feature-reading. As if explaining to a friend

### Fields

- `visualText` (string): All text that appears on screen in this scene. Copywriter fills this
- `narrativeIntent` (string): What the voice should COMMUNICATE (not say literally). Copywriter fills this
- Both fields are stripped before render (internal metadata only)

## 3. Scene Library Expansion (ProductShort)

### Scenes to port from ClaudeCodeTutorial custom library

These exist as landscape custom scenes. They need vertical (1080x1920) adaptations with ProductShort theming.

| Scene type         | Source component     | Purpose                        | Key adaptations for vertical                                    |
| ------------------ | -------------------- | ------------------------------ | --------------------------------------------------------------- |
| `quote`            | QuoteScene           | Testimonials, authority claims | Larger quote text (36px), centered vertical layout              |
| `stat-reveal`      | StatRevealScene      | Impact number with counter     | Counter number 120px, full-width progress bar                   |
| `comparison-table` | ComparisonTableScene | Us vs competition              | 2-column max (screen too narrow for more), larger cells         |
| `problem-solution` | ProblemSolutionScene | Narrative tension              | Vertical stack (problem top, solution bottom), taller connector |
| `timeline`         | TimelineScene        | Process, history               | Same vertical layout, works naturally                           |
| `bullet-slide`     | BulletSlideScene     | Feature list with icons        | Wider items (full width), larger icons                          |

### New scenes to create

#### `screenshot`

Device-framed screenshot or image with animated annotations.

```json
{
  "type": "screenshot",
  "title": "Gestiona tu poliza desde el movil",
  "imageSrc": "/images/app-screenshot.png",
  "device": "phone",
  "annotations": [{ "x": 50, "y": 30, "text": "Dashboard intuitivo", "position": "right" }],
  "durationInSeconds": 8
}
```

Props:

- `title` (string, optional)
- `imageSrc` (string): Path to screenshot image
- `device` ("phone" | "desktop"): Frame style
- `annotations` (array, optional): Positioned callouts over image
- `durationInSeconds`: 5-15

Visual: Device mockup frame (phone bezel or browser chrome), image inside, numbered annotations that appear with stagger. Phone mockup for vertical, browser for landscape.

#### `split-stat`

Before/after comparison with two big numbers.

```json
{
  "type": "split-stat",
  "title": "Ahorro real",
  "left": { "value": "340", "suffix": "€", "label": "Otros seguros" },
  "right": { "value": "189", "suffix": "€", "label": "Linea Directa" },
  "highlight": "right",
  "durationInSeconds": 7
}
```

Props:

- `title` (string, optional)
- `left` ({value: string, suffix?: string, prefix?: string, label: string})
- `right` ({value: string, suffix?: string, prefix?: string, label: string})
- `highlight` ("left" | "right"): Which side to emphasize
- `durationInSeconds`: 5-10

Visual: Two large numbers side by side with animated counters. Highlighted side has primary color + scale emphasis. Arrow or VS between them.

### Schema changes

`src/compositions/ProductShort/schema.ts`:

- Add 8 new scene schemas to the `ProductShortSceneSchema` union
- Each with `.merge(DirectionSceneFieldsSchema)` for timing/beats support
- Add `visualText` and `narrativeIntent` as optional strings to a shared scene fields schema

### Registry

ProductShort scenes are dispatched directly in the composition (no custom registry needed — they're first-class scene types in the schema union).

## 4. Transitions

### Activation

Migrate from `<Series>` to `<TransitionSeries>` from `@remotion/transitions` in both compositions.

### Transition types

| Type    | When                                  | Duration (frames) | Effect                |
| ------- | ------------------------------------- | ----------------- | --------------------- |
| `fade`  | Same narrative block                  | 15-20 (0.5-0.7s)  | Cross-dissolve        |
| `slide` | Block change (hook->context)          | 10-15 (0.3-0.5s)  | Directional slide     |
| `wipe`  | Before dramatic moment (pricing, CTA) | 12-18 (0.4-0.6s)  | Horizontal wipe       |
| `none`  | After impact scene (stat-reveal)      | 0                 | Hard cut for emphasis |

### Implementation

- Default transition: `fade` (15 frames)
- Director adds `transitionType` field to each scene during direction phase
- Composition reads `transitionType` and applies corresponding `@remotion/transitions` presentation
- `TransitionConfig` already exists in the schema — extend it with per-scene type

### Director rules

- Never use the same transition 3 times in a row
- `wipe` reserved for max 2 uses per video
- First scene always enters with `none` or `fade`
- CTA always enters with `wipe` or `slide`

## 5. Prompt Rewrites

### copywriter.md — Changes

**Add:**

- Duration enforcement section with min/max per format
- Narrative structure (5-block template)
- Documentation for all 12 ProductShort scene types (4 existing + 8 new)
- `visualText` and `narrativeIntent` internal fields per scene
- Scene selection guide: which scene for which narrative intent
- Copy density rules: hero max 8 words, benefit items max 12, CTA max 5
- Rule: never repeat scene type consecutively
- Rule: minimum 6 scenes for any video

**Modify:**

- Creative rules: add "write like creative director, not feature list"
- Hook section: stronger guidance on provocation vs generic intros

### director.md — Changes

**Add:**

- `transitionType` field per scene
- Transition rules (which type when, never 3x same)
- Voiceover timing: voice before or after visual, never exactly sync
- Intensity curve: alternate high/medium/low across video
- Scene-specific direction patterns for all 12 types
- Mandatory silence markers for impact scenes

### audio_planner.md — Changes

**Add:**

- Show vs tell protocol (full rules)
- Read `visualText` and `narrativeIntent` from each scene
- Prohibition on repeating visible text
- Mandatory 2s silence on hero, pricing, stat-reveal
- Tone guidance: conversational, empathetic, not robotic
- Text density: max 2 short sentences per scene, max 15 words per sentence

**Remove:**

- Any guidance that encourages describing what's on screen
